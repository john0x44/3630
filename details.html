Skill Details

We can explore specific details about databases and SQL focusing on key terminology and their explanations.

Detailed Explanation With SQL

  • Relational Database: relational database means where data is organized into tables which are made up of rows and columns. The tables are related to each other through common data.
  • Record: A single row in a database table that holds all the data for one item.
  • Schema: A way to define the structure of a database.
  • Normalization: Organizing a database so there is no repeated data.
  • Tables: The database is broken up into tables can use ids to connect to other tables in other words there are relations between them. A table contains rows and columns, in a database colums are reffered to fields and rows are reffered to as records. Columns are broken down because they can be easily accessed. Tables can relate to each other via primary keys or references
  • Primary Keys: primary key minimum number of columns needed to identify a record or in other words a unique identifier for each record in a table which helps to uniquely identify and access data
  • Foreign Keys: Links one table to another by referencing the primary key in the other table.
  • Database Diagrams: A way to view how database is connected for example how tables have relationships between them.
  • Datatypes: There might be a specific datatype for each column name.
  • Queries: A request for data from a database.
  • Filtering: Filtering simply means we are selecting specific rows from a table based on a condition or in other words narrowing down results to only show data that you need

Techniques

Queries

  • SELECT: Retrieves all data from a table
    SELECT * FROM table1
  • INSERT: Adds new data into a table
    INSERT INTO table1 (name, age) VALUES ('John', 24)
  • UPDATE: Changes existing data in a table
    UPDATE table1 SET age = 31 WHERE name = 'John'
  • DELETE: Removes data from a table
    DELETE FROM table1 WHERE name = 'John'

Filtering

  • Example: Retrieve all customers from New York
    SELECT * FROM table1 WHERE State = 'NY'
  • NOT Operation: Get customers not from New York
    SELECT * FROM table1 WHERE State <> 'NY'
  • OR Operation: Get customers from New York or Washington
    SELECT * FROM table1 WHERE State = 'NY' OR State = 'WA'

Joins

Joins combine data from multiple tables. For example:

If we want to get columns from the table named orders such as order id, order date, order total and include a customer name and customer phone column we can use join on the primary key referencing the table customers that has a primary key called customer id which has columns with costumer name and customer phone we can utilize a JOIN query for example

SELECT OrderID, OrderDate, OrderTotal, CustomerName, Phone
FROM Orders
JOIN Customers
ON Orders.CustomerID = Customers.CustomerID;

This type of join is called an inner join which simply combines rows from two tables based on a matching condition here we are matching with customer id
